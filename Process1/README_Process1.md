1. การทดลองระยะเริ่มต้น (Initial Implementation)
---------------------------------------------
  แนวคิดแรก: ใช้ Pipeline พื้นฐาน ได้แก่ Grayscale → Gaussian Blur → Otsu's Thresholding → Find Contours
  ผลการทดสอบ: ผลลัพธ์ยังไม่ถูกต้อง นำไปสู่การปรับปรุงร่วมกับคำแนะนำของทีม(พี่ไอซ์) โดยเพิ่มโค้ดแสดงผลลัพธ์แยกแต่ละขั้นตอนเพื่อหาจุดบกพร่อง 
               และจำกัดพื้นที่การนับเฉพาะในจานเพาะเชื้อ (ROI)

2. ปัญหาที่พบจากการทดสอบ (Key Issues)
-------------------------------------
  จุดนับไม่ครบ: เกิดจากการกำหนดพื้นที่ครอบจานเพาะเชื้อที่กินพื้นที่ขอบเข้ามาด้านในมากเกินไป ทำให้จุด Colony บริเวณขอบจานหายไป
           เกิดจุดหลอก (Phantom Points): การใช้คำสั่ง Top-Hat Transform ตามด้วย Subtle Erosion & Dilation 
           เจอปัญหาแสงเงาบริเวณขอบจานเพาะเชื้อ ทำให้เกิดจุดหลอกขึ้นจำนวนมาก

3. การทดลองและแนวทางการแก้ไข (Iterative Solutions)
--------------------------------------------------
  แนวทางที่ 1: กลบเงาขอบจาน
    วิธีแก้: ใช้ Otsu's Thresholding ทันทีหลังทำ Top-Hat เพื่อตัดแสงเงาขอบจานทิ้งก่อนประมวลผลขั้นต่อไป
    ผลลัพธ์: กำจัด Phantom Points ได้สำเร็จ แต่เกิดปัญหาใหม่คือ จุด Colony ขยายใหญ่ขึ้นจนจุดที่อยู่ใกล้กันกลืนเป็นเนื้อเดียวกัน
  
  แนวทางที่ 2: แยกจุดที่ติดกันด้วยรูปทรง (Contour Shape Analysis)
    วิธีแก้: พยายามหาส่วนโค้งเพื่อจำแนกองค์ประกอบภาพและแยกจุดที่ติดกัน
    ผลลัพธ์: ล้มเหลว ไม่สามารถนำมาใช้งานจริงได้

  แนวทางที่ 3: ปรับขนาดจุดเพื่อสร้างระยะห่าง (สําเร็จ)
    วิธีแก้: ปรับกระบวนการ Otsu's Thresholding ให้ลดขนาดของจุดลงประมาณ 25%
    ผลลัพธ์: จุดที่เคยติดกันสามารถแยกออกจากกันได้อย่างชัดเจนและนับจำนวนได้ถูกต้อง


4. การเพิ่มประสิทธิภาพการนับในปัจจุบัน (Optimized Solution)
----------------------------------------------------
  เพื่อเพิ่มความแม่นยำในการคำนวณพื้นที่และนับจำนวน ได้พัฒนาต่อยอดไปใช้วิธี Ray-Casting For Boundary Detection มีหลักการทำงานดังนี้:
    1.พล็อตจุดศูนย์กลางของ Colony
    2.ยิงลำแสง (Ray) ออกไปรอบทิศทางเพื่อตรวจจับขอบ โดยดูจากความต่างของสีพื้นผิว
    3.นำความยาวของ Ray ที่ใกล้เคียงกันมาหาค่าเฉลี่ยเพื่อเป็นรัศมี(r)
    4.คำนวณพื้นที่วงกลมโดยประมาณเพื่อใช้เป็นตัวแทนของจุด Colony
